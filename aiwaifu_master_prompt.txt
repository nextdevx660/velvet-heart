# 🌸 AI Waifu SaaS Platform — Full Build Prompt

## Project Overview
Build a full-stack AI Waifu SaaS web application where users can sign up,
pick an AI female character (waifu), and have flirty/romantic/friendly
conversations with them. Each character has a unique personality powered
by an LLM. The platform is freemium — free users get limited messages,
paid users get unlimited access.

---

## 🛠️ Tech Stack (All Free Tier)
- Frontend: Next js [JAVASCRIPT] Tailwind CSS + shadcn/ui
- Backend: Node.js + Express
- AI: Groq API (model: llama3-70b-8192) — FREE
- Database + Auth: firebase (PostgreSQL + Supabase Auth)
- Hosting: Vercel (frontend) + Render (backend)
- Payments: paypal (free to set up)

---



## 🗄️ Supabase Database Schema

### Table: profiles
- id          uuid (references auth.users)
- username    text
- plan        text DEFAULT 'free'  -- 'free' or 'pro'
- created_at  timestamp

### Table: characters
- id            uuid
- name          text
- description   text
- personality   text
- avatar_url    text
- tag           text  -- e.g. "shy", "playful", "tsundere"

### Table: conversations
- id              uuid
- user_id         uuid (references profiles.id)
- character_id    uuid (references characters.id)
- created_at      timestamp

### Table: messages
- id                uuid
- conversation_id   uuid (references conversations.id)
- role              text  -- 'user' or 'assistant'
- content           text
- created_at        timestamp

### Table: usage
- id          uuid
- user_id     uuid
- date        date
- count       integer DEFAULT 0

## 🔌 Backend API Endpoints

### POST /api/chat
Request body:
{
  "character_id": "uuid",
  "conversation_id": "uuid",
  "message": "Hey Sakura!",
  "user_id": "uuid"
}

Logic:
1. Verify Supabase JWT from Authorization header
2. Check usage table — if free user has 20+ messages today, return 403
3. Fetch last 10 messages from this conversation (for memory)
4. Build Groq API call:
   - system: character's system prompt
   - messages: last 10 messages + new user message
5. Save user message + AI reply to messages table
6. Increment usage count for today
7. Return AI reply

Response:
{
  "reply": "Hiii~! I was just thinking about you! (≧◡≦)",
  "messages_used": 5,
  "limit": 20
}

### GET /api/characters
Returns all characters from Supabase characters table

### GET /api/conversation/:character_id
Returns or creates a conversation for this user + character

### POST /api/stripe/create-checkout
Creates a Stripe checkout session for Pro plan

### POST /api/stripe/webhook
Listens for Stripe payment success → updates user plan to 'pro' in profiles

## 🔐 Auth Flow
1. User signs up / logs in via Supabase Auth
2. On first login, create a row in profiles table
3. Every API request sends Supabase JWT in Authorization: Bearer <token> header
4. Backend middleware verifies JWT using Supabase admin client
5. Extract user_id from verified token and use it for all DB queries

---

## 💳 Payments Flow
1. User clicks "Upgrade to Pro"
2. Frontend calls POST /api/stripe/create-checkout
3. Backend creates Stripe Checkout Session → returns URL
4. User is redirected to Stripe checkout page
5. On success, Stripe fires webhook to POST /api/stripe/webhook
6. Backend updates profiles.plan = 'pro' for that user
7. User now has unlimited messages

## ✅ Rules for Codex / AI Coding Assistant
- Always use async/await, never callbacks
- Always handle errors with try/catch and return proper error messages
- Always validate request body before processing
- Use Supabase client from @supabase/supabase-js
- Use Groq SDK: groq-sdk npm package
- Keep React components small and reusable
- Use Tailwind for ALL styling, no custom CSS files
- Never hardcode API keys, always use .env variables
- Add a comment to every function explaining what it does
- Use uuid package for generating IDs where needed
- Always return consistent JSON responses from the backend
