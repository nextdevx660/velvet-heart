# 🌸 AI Waifu SaaS Platform — Full Build Prompt (Next.js + Firebase + Vercel)

## Project Overview
Build a full-stack AI Waifu SaaS web application using Next.js (JavaScript).
Users can sign up, pick an AI female character (waifu), and have
flirty/romantic/friendly conversations with them. Each character has a
unique personality powered by Groq AI. The platform is freemium —
free users get limited messages, paid users get unlimited access.

---

## 🛠️ Tech Stack (All Free Tier)
- Framework:  Next.js (JavaScript) — frontend + backend API routes in one project
- Styling:    Tailwind CSS + shadcn/ui
- AI:         Groq API (model: llama3-70b-8192) — FREE
- Database:   Firebase Firestore (NoSQL, free tier)
- Auth:       Firebase Authentication (free tier)
- Hosting:    Vercel (free tier) — deploys both frontend and API routes
- Payments:   Stripe (free to set up, takes % per transaction only)

## 🔥 Firebase Firestore Collections

### Collection: users
Document ID: firebase auth uid
Fields:
- uid          string
- email        string
- username     string
- plan         string  ('free' or 'pro')
- createdAt    timestamp

### Collection: characters
Document ID: character slug (e.g. "sakura")
Fields:
- id           string
- name         string
- description  string
- tag          string   (e.g. "playful", "tsundere", "shy")
- avatarUrl    string
- isFree       boolean  (true = available on free plan)

### Collection: conversations
Document ID: auto-generated
Fields:
- userId        string
- characterId   string
- createdAt     timestamp
- memory        string  (pro only — summary of user facts)

### Collection: messages
Document ID: auto-generated
Fields:
- conversationId  string
- role            string  ('user' or 'assistant')
- content         string
- createdAt       timestamp

### Collection: usage
Document ID: userId_date (e.g. "abc123_2024-01-15")
Fields:
- userId   string
- date     string
- count    number

## 🔐 Firebase Auth Setup
- Enable Email/Password authentication in Firebase console
- Enable Google Sign-In as a bonus option
- Use Firebase Auth on client side for login/signup
- Use Firebase Admin SDK on server side (API routes) to verify tokens

### Client Side (lib/firebase.js)
import { initializeApp } from 'firebase/app'
import { getAuth } from 'firebase/auth'
import { getFirestore } from 'firebase/firestore'

const firebaseConfig = {
  apiKey: process.env.NEXT_PUBLIC_FIREBASE_API_KEY,
  authDomain: process.env.NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN,
  projectId: process.env.NEXT_PUBLIC_FIREBASE_PROJECT_ID,
  storageBucket: process.env.NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET,
  messagingSenderId: process.env.NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID,
  appId: process.env.NEXT_PUBLIC_FIREBASE_APP_ID,
}

const app = initializeApp(firebaseConfig)
export const auth = getAuth(app)
export const db = getFirestore(app)

### Server Side (lib/firebaseAdmin.js)
import admin from 'firebase-admin'

if (!admin.apps.length) {
  admin.initializeApp({
    credential: admin.credential.cert({
      projectId: process.env.FIREBASE_PROJECT_ID,
      clientEmail: process.env.FIREBASE_CLIENT_EMAIL,
      privateKey: process.env.FIREBASE_PRIVATE_KEY.replace(/\\n/g, '\n'),
    }),
  })
}

export const adminDb = admin.firestore()
export const adminAuth = admin.auth()

## 🔌 API Routes (Next.js Route Handlers)

### POST /api/chat  (app/api/chat/route.js)
Request body:
{
  "characterId": "sakura",
  "conversationId": "abc123",
  "message": "Hey Sakura!"
}

Logic:
1. Get Authorization header → verify Firebase ID token using adminAuth
2. Extract uid from decoded token
3. Check usage doc (userId_todayDate) in Firestore
   - If free user and count >= 20 → return 403 with upgrade message
4. Fetch last 10 messages from messages collection for this conversationId
5. Get character system prompt from lib/characters.js using characterId
6. Call Groq API:
   - system prompt: character prompt
   - messages: last 10 messages + new user message
7. Save user message to Firestore messages collection
8. Save AI reply to Firestore messages collection
9. Increment usage count in Firestore usage collection
10. Return AI reply

Response:
{
  "reply": "Hiii~! I was just thinking about you! (≧◡≦)",
  "messagesUsed": 5,
  "limit": 20
}

### GET /api/characters  (app/api/characters/route.js)
- No auth required
- Returns all documents from Firestore characters collection

### POST /api/stripe/create-checkout  (app/api/stripe/create-checkout/route.js)
- Verify Firebase token
- Create Stripe checkout session for Pro plan
- Return checkout URL

### POST /api/stripe/webhook  (app/api/stripe/webhook/route.js)
- Verify Stripe webhook signature
- On payment success → update users doc plan field to 'pro'

## 🔐 Auth Flow
1. User signs up on /signup using Firebase Auth (createUserWithEmailAndPassword)
2. On signup success, create a document in Firestore users collection with plan: 'free'
3. User logs in on /login (signInWithEmailAndPassword)
4. Firebase gives a JWT ID token on the client
5. Every API call sends this token: Authorization: Bearer <idToken>
6. API route calls adminAuth.verifyIdToken(token) to authenticate
7. Use the uid from decoded token for all Firestore queries

### useAuth.js hook
- Wraps onAuthStateChanged from Firebase
- Returns: { user, loading }
- Use this in every protected page to redirect if not logged in

---

## 💳 Payments Flow
1. User clicks "Upgrade to Pro" in UpgradeModal or /pricing page
2. Frontend gets Firebase ID token and calls POST /api/stripe/create-checkout
3. Backend creates Stripe Checkout Session with price ID
4. Returns { url } to frontend
5. Frontend does window.location.href = url to redirect to Stripe
6. User pays on Stripe hosted page
7. Stripe sends webhook to POST /api/stripe/webhook
8. Backend updates Firestore users doc: plan = 'pro'
9. User is redirected to /dashboard with pro access

## 🧠 Memory Feature (Pro Only)
- After every 5 messages, call Groq with this prompt:
  "Summarize key facts about this user based on the conversation. Be brief."
- Store the summary in the conversations Firestore doc under memory field
- On every new chat request, fetch the conversation doc
- If memory exists, inject it into the system prompt:
  "What you remember about this user: [memory here]"

## ✅ Rules for Codex / AI Coding Assistant
- This is a Next.js JavaScript project, NOT TypeScript
- Use App Router (app/ directory), NOT pages/ directory
- All API routes use Next.js Route Handlers (route.js), NOT express
- Always use async/await, never callbacks
- Always wrap everything in try/catch and return NextResponse.json()
- Use Firebase Admin SDK only in API routes (server side)
- Use Firebase Client SDK only in components and hooks (client side)
- Never import firebase-admin in client-side files
- Use Tailwind for ALL styling, zero custom CSS
- Never hardcode API keys, always use process.env
- Add a comment above every function explaining what it does
- Keep components small — one component per file
- Always return consistent JSON from API routes:
  { success: true, data: ... } or { success: false, error: '...' }
